Machine Learning Workshop ** Viewport ** -------------- - It refers to the part of the document you're viewing which is currently visible in its window. - Content outside the viewport is not visible onscreen until scrolled into view. - Layout viewport ==> The layout viewport is the viewport into which the browser draws a web page. Essentially, it represents what is available to be seen - Visual viewport ==> Represents what is currently visible on the user's display device. [window.visualViewport] ** Viewport metadata ** ----------------------- - helps site responsiveness, enabling content to render well by default, no matter the viewport width. - it enables controlling a viewport's size and scale, and prevents the site's content from being sized down to fit a 960px site onto a 320px screen, it is definitely recommended. - Above html means "make the site responsive, starting by making the width of the content the width of the screen" - you can set zoom and scalability, but they both default to accessible values. If need to be explicit, include : The basic properties of the "viewport" tag include: * width -> Controls the size of the viewport width=600 (px) width=device-width (which is 100vw, 100% of viewport width) 1 < width < 10000 * height -> Controls height of the viewport height=400 (px) width=device-height (which is 100vh, 100% of viewport height) 1 < 10000 < 10000 * initial-scale -> Controls the zoom level when the page is first loaded 0.1 < initial-scale < 10 [ Default: 1 ] * minimum-scale -> Controls how much "zoom out" is allowed on the page 0.1 < initial-scale < 10 [ Default: 0.1 ] * maximum-scale -> Controls how much "zoom in" is allowed on the page Any value less than 3 fails accessibility. 0.1 < initial-scale < 10 [ Default: 10 ] * user-scalable -> Controls whether zoom in and zoom out actions are allowed on the page. 0, 1, yes, no [ Default: 1 (yes)] Setting to 0 (no), is against WCAG (Web Content Accessibility Guidelines) https://www.w3.org/TR/WCAG21/#wcag-2-layers-of-guidance { WCAG requires a minimum of 2× scaling; however, the best practice is to enable a 5× zoom. } * interactive-widget -> Specifies the effect that interactive UI widgets, such as a virtual keyboard, have on the page's viewports. resizes-visual, resizes-content, or overlays-content.
